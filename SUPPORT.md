# Support

## Getting Help

Use the repository documentation first:

- `README.md` for setup and scripts
- `CONTRIBUTING.md` for workflow and commit expectations
- `SECURITY.md` for private vulnerability handling

## When Opening an Issue

Include:

- the exact page, route, or API involved
- what you expected to happen
- what happened instead
- relevant screenshots or logs
- steps to reproduce
- whether the issue is local-only or also on GitHub-hosted environments

## Local Setup Checklist

Before reporting a problem, confirm:

- dependencies are installed
- Prisma client has been generated
- database has been pushed and seeded
- required env vars are present
- `npm run typecheck` passes

## Not Supported Here

This repository should not be used to disclose secrets, payment credentials, or public vulnerability details.
